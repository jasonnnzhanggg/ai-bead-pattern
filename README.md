# AI Bead Pattern

把客户上传的图片转换成可编辑的 MARD 拼豆模板。当前版本是本地优先的浏览器应用：图片、项目和进度都保留在用户浏览器内。

## 支持格式

- 上传图片：PNG、JPEG、WebP
- 成品输出：项目 JSON、PNG 预览、SVG、PDF 色号清单和打印校准页
- 色卡：MARD 291 色快照，来源记录为 `pixel-beads-2026-07-06`

## 本地处理和保存

- 图片不会上传到服务器。
- 项目会保存在浏览器本地存储中，用于刷新后恢复当前模板和拼豆进度。
- 浏览器清除站点数据后，本地项目也会被删除。
- 建议客户在重要项目完成后导出项目 JSON 作为备份；换电脑或清空浏览器数据前，应先导出项目文件。

## 拼豆流程

1. 上传主体清晰的图片。
2. 选择豆子规格和板型，例如 25×25、29×29、30×30、50×50 或 26×20cm 物理尺寸。
3. 选择推荐版本后进入编辑模式。
4. 逐格修正色号。
5. 进入拼豆模式，在底部选择色号；同色格会高亮，其它格变透明。
6. 点击格子标记完成进度。
7. 导出 PDF 或其它格式用于制作。

## 尺寸说明

物理尺寸由豆子 pitch 计算，实际成品会受品牌豆子、公差、熨烫收缩和底板误差影响。PDF 内含 100mm 校准尺，打印时应使用 100% 原尺寸。

## 本地 AI 前景识别

当前代码已经有本地 AI 分割适配层、进度、取消和手动选择主体 fallback。真实 ONNX/Transformers 模型文件应从应用同源加载并缓存；失败时保留原图，不直接删除客户图片，也不直接决定最终 MARD 色号。

## 开发命令

```bash
npm install
npm run dev
npm test
npm run build
```

端到端测试使用 Playwright：

```bash
npx playwright test
```

本地环境如果缺少 Playwright 浏览器，需要先安装浏览器；本仓库建议把浏览器缓存放在项目目录并忽略提交。
